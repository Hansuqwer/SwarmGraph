🔴 Critical Findings & Fixes

CRITICAL C1 · WorkflowState Missing ConfigDict Hardening
WorkflowState had no ConfigDict. Unknown checkpoint fields were silently ignored (deserialization attack surface). Field mutations bypassed validators. A compromised checkpoint JSON could inject unknown fields with no error.
Fix: Added ConfigDict(extra='forbid', validate_assignment=True) to WorkflowState. Tests: test_workflow_state_hardening.py::TestExtraForbid
CRITICAL C2 · Non-Atomic Checkpoint Writes
checkpoint_path.write_text() is not atomic. A crash mid-write leaves a corrupt JSON file. On resume, this raised a raw json.JSONDecodeError, not the clean CheckpointNotFound.
Fix: Atomic write via tempfile.mkstemp() + os.replace(). Corrupt files raise CheckpointCorrupt. Tests: test_checkpoint_atomic_write.py
CRITICAL C4 · MemoLesson Shell Metachar Denylist Incomplete
Original regex r"[;&|<>\\`$]" missed: ! (history expansion), ( ) (subshell), { } (brace expansion), \n \r (newline injection), * ? (glob), ^ ~.
Fix: Expanded to r'[;&|<>\\`$!()\{\}\n\r*?^~]' + glob character allowlist on file_glob. Tests: test_memo_lesson_hardening.py::TestUnsafeSummaries
CRITICAL C6 · TokenUsage Accepts Negative Values
TokenUsage(input_tokens=0, output_tokens=0) accepted negative integers. A buggy gateway could record input_tokens=-1, silently corrupting usage accounting and cost tracking.
Fix: Added Field(default=0, ge=0) to both fields. Tests: test_workflow_state_hardening.py::TestTokenUsageBounds
CRITICAL C9 · fail_closed() Can Leave failure_cause=None
Unexpected exceptions fell through all isinstance() checks without setting failure_cause. Monitoring and alerting could not distinguish between known and unknown failure modes.
Fix: Added catch-all branch: unknown exceptions → failure_cause='unknown', status='failed'. Tests: test_fail_closed_comprehensive.py
CRITICAL C10 · history Is Untyped list[dict[str, Any]]
History entries had no schema. Entries with kind="shell", "agent", "patch_validation" etc. were unvalidated dicts — impossible to validate, evolve safely, or introspect reliably in tests.
Fix: Introduced typed HistoryEntry discriminated union covering 6 entry kinds. Bounded lists with append_history() helper.

🟠 High Priority Findings & Fixes

HIGH C7 · Unbounded history / errors Lists
_cap_state_history() was called in LangGraphRuntime._export_result but NOT in the legacy path or LocalCheckpointStore.save(). Long-running workflows could exhaust memory/storage.
Fix: Model validator in WorkflowState caps lists on every construction. append_history() and add_error() helpers enforce caps on mutation.
HIGH C8 · repo_root / thread_id / task Not Validated
repo_root: str accepted empty strings, whitespace, and paths with .. traversal. thread_id and task accepted empty strings.
Fix: @field_validator on all three fields. Tests: test_workflow_state_hardening.py::TestRepoRootValidation
